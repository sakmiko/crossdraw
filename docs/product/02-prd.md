# Crossdraw 产品需求文档（PRD）

**状态：** Accepted  
**版本：** 0.1.0  
**日期：** 2026-07-15  
**依据：** 内部文档包 + RoadGee 公开功能审计（`research/01-roadgee-feature-audit.md`）+ 竞品补强（`research/02-market-landscape.md`）

## 1. 背景

市面以 RoadGee 为代表的浏览器 SaaS 已验证「交叉口参数化出图」需求，但存在：账号与次数、云端保密顾虑、闭源不可审计、方案数硬限制（如 3×3×3）、免费版水印等约束。

Crossdraw 以 **本地免费开源** 提供同等工作流，并在联动可靠性、规范工程化、可扩展引擎上建立优势。

## 2. 目标与非目标

### 2.1 目标

| ID | 目标 | 度量 |
|----|------|------|
| G1 | 单交叉口全套静态出图 | 渠化/流量/信号/断面可导出 |
| G2 | 参数全图联动 | 关键参数变更后相关图 100% 自动更新 |
| G3 | 本地工程文件 | `.rtp` 保存/加载/自动保存/崩溃恢复 |
| G4 | 规范友好 | 默认值标注 CJJ/GB 来源；OK/WARN 提示 |
| G5 | 可对接下游 | PNG/SVG/DXF；分析表；仿真中间格式路线图 |

### 2.2 非目标（见路线图明细）

- V1 不做完整绿波优化产品线  
- V1 不做多交叉口路网配时协调  
- V1 不做实时协同编辑  
- 不做第三方商业软件破解  

## 3. 用户故事（V1 核心）

1. 作为设计人员，我画/生成一个十字交叉口，以便快速得到渠化底图。  
2. 作为设计人员，我调整展宽与车道功能，画布与标注自动更新。  
3. 作为设计人员，我输入转向流量，得到流率箭头图。  
4. 作为信号人员，我编辑相位与周期，得到 Ring-Barrier / 相位图。  
5. 作为设计人员，我为进口道生成横断面简图。  
6. 作为用户，我保存 `.rtp` 并导出 PNG/SVG 用于报告。

## 5. 体验需求

| ID | 需求 |
|----|------|
| UX1 | 三栏布局：左项目/工具，中画布，右参数+模式 |
| UX2 | 模式：渠化 | 流量 | 信号 | 横断面 |（分析 V1.1） |
| UX3 | 常用参数默认展开，高级折叠 |
| UX4 | 合规状态 `[OK]` `[WARN]` `[BLOCK]`（BLOCK 仅严重错误，默认不阻止导出） |
| UX5 | 界面无 emoji，统一图标系统 |

## 6. 质量属性

| 属性 | 要求 |
|------|------|
| 性能 | 单路口编辑交互 60fps 目标（常规复杂度） |
| 可靠 | 自动保存间隔 ≤ 30s；崩溃可恢复 |
| 可测 | domain 引擎单测覆盖核心几何与分析 |
| 可访问 | 关键操作具备快捷键（见 specs） |
| 安全 | 本地文件；无强制遥测；可选匿名统计默认关 |

## 7. 商业与许可

- 开源版：GPLv3，个人与机构免费使用  
- 可选未来：企业支持合同、培训、定制导出（不关闭开源核心）
