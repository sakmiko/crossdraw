# Crossdraw 产品愿景

**状态：** Accepted  
**版本：** 0.1.0  
**日期：** 2026-07-15

## 1. 一句话

**Crossdraw 是本地优先、免费开源的交叉口设计图纸生成器：参数一次输入，渠化 / 流量 / 信号 / 横断面自动联动出图。**

## 2. 本质与非目标

### 2.1 是

- 交通工程师的**方案表达与出图工作站**
- **参数化对象**系统（道路 / 进口道 / 车道组 / 相位是数据，不是散线）
- 可归档的工程文件（`.rtp`），适合设计院与课程作业

### 2.2 不是

- AutoCAD / Civil 3D 替代品  
- Vissim / Synchro 级仿真与优化全家桶（可对接，不取代）  
- 多用户实时协作云 BIM  
- 闭源商业 SaaS 的资源拷贝

## 3. 要解决的问题

传统流程割裂：

```
CAD 画路 → Excel 算流量 → Visio/PPT 画相位 → 手工改横断面 → 再对仿真
```

痛点：改车道宽度要改四处；规范默认靠人脑；云端按次扣费与保密冲突。

Crossdraw 的答案：

```
一个项目文件 → 一套参数 → 多类图纸 + 可选分析 → 导出 CAD/图片/仿真中间格式
```

## 4. 目标用户

| 用户 | 核心诉求 |
|------|----------|
| 交通/市政设计院工程师 | 快出图、方案对比、可归档、可进 CAD |
| 信号优化工程师 | 相位方案表达、自动配时、评价表 |
| 高校师生 | 教学演示、课设、论文插图、免费 |

## 5. 设计原则

1. **生成优先** — 先稳定生成，再优化分析  
2. **所见即所得** — 画布 = 导出预览  
3. **对象即数据** — 全图联动，禁止死线主导流程  
4. **国标默认、专家可覆盖** — CJJ/GB 推荐值可溯源  
5. **事无巨细的辅助** — 一键修复、撤销、自动保存  
6. **本地信任** — 无强制账号、无次数墙、无水印（开源版）  
7. **克制界面** — 专业工具感；无 emoji；80/20 披露  

## 6. 产品支柱（不可妥协）

| 支柱 | 说明 |
|------|------|
| 方案树 | 渠化 ⊃ 流量 ⊃ 信号（心智对齐主流工具，数量限制可放宽） |
| 参数驱动画布 | 右栏改数，几何重算 |
| 国标预设 | 默认即合规倾向 |
| `.rtp` 文件 | 单一真相来源 |
| 引擎可测 | 几何/信号/分析纯函数 + golden fixtures |

## 7. 成功标准（产品级）

- 新用户 **15 分钟内** 完成：十字路口 → 渠化 → 流量 → 相位 → 导出 PNG  
- 修改进口车道数后，**渠化图与横断面无需手工重画**  
- 项目断电恢复后，**工作不丢**  
- 无网络可完成 V1 主路径  

## 8. 品牌与传播

- 产品名：**Crossdraw**  
- 定位语：本地交叉口出图工作站  
- 禁止：「RoadGee 克隆 / 破解版」等表述
