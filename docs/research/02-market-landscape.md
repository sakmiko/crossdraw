# 市场相关软件与网页调研（取长补短）

**状态：** Accepted（初稿，待子代理结果可修订增补）  
**日期：** 2026-07-15

## 1. 调研目的

在 RoadGee 主对标之外，吸收仿真、配时、CAD、开源地图等工具的长处，避免功能片面，同时守住「出图生成器」边界。

## 2. 工具矩阵

| 工具 | 类型 | 核心能力 | 优势 | 可借鉴 | 不该照搬 |
|------|------|----------|------|--------|----------|
| **RoadGee** | Web SaaS | 渠化/流量/信号/断面/绿波/分析/Vissim | 工作流完整、出图快 | 方案树、参数面板、公开分析口径 | 扣次、闭源、3 上限 |
| **Synchro** | 桌面商业 | 配时、时段、协调、HCM 报告 | 行业标准配时 | 报告结构、相位时间线 | 重许可、学习曲线 |
| **PTV Vissim** | 桌面商业 | 微观仿真 | 仿真精度 | 导出字段映射、路径/信号机概念 | 不做完整仿真引擎 |
| **PTV Vistro** | 桌面 | 交叉口分析与配时 | 与 PTV 生态一体 | 分析报告版式 | 商业绑定 |
| **SIDRA Intersection** | 桌面 | 交叉口分析、圆岛 | 分析深度 | 车道组概念、圆岛远期 | 分析器定位不同 |
| **TRANSYT** | 桌面 | 路网信号优化 | 网络优化 | V3 路网阶段概念 | V1 不做网络 |
| **HCS (HCM)** | 桌面 | HCM 方法计算 | 方法论权威 | 公式与术语对齐 | 全套 HCM 模块 |
| **AutoCAD Civil 3D** | 桌面 | 道路 BIM/平纵横 | 行业出图终点 | DXF 图层纪律、标注 | 不做 BIM |
| **国内市政 CAD 插件** | 插件 | 国标图框、交规符号 | 本土符号 | 国标图层命名、图框 | 依附 CAD |
| **QGIS / OSM** | 桌面/数据 | 地图与路网数据 | 开源地理 | V2 底图、坐标参考 | 不做 GIS 全家桶 |
| **draw.io / Excalidraw** | Web | 示意速绘 | 轻 | 快捷键、极简 UI | 无交通语义 |
| **StreetMix** | Web | 断面拼装趣味 | 断面组件 UX | 横断面组件拖拽心智 | 非工程精度 |
| **AASHTO/绿皮书工具** | 参考 | 设计标准 | 标准 | 默认值表结构 | 地域以国标优先 |
| **SUMO** | 开源仿真 | 微观/中观 | 开源 | 远期导出对接 | 不内嵌重仿真 |

## 3. 功能补全建议（相对只抄 RoadGee）

| 补全项 | 来源启发 | 纳入版本 |
|--------|----------|----------|
| 车道组（Lane Group）一等公民 | SIDRA/HCM/Synchro | V1 数据模型即支持 |
| 报告式分析表（不仅热力） | Synchro/Vistro | V1.1 |
| DXF 图层与图框规范 | Civil/国内插件 | V1 |
| 模板库（十字/T/畸形简化） | 多工具 | V1 |
| 命令面板与快捷键密度 | 现代编辑器 | V1–V1.2 |
| 坐标与北向、比例尺出图 | CAD | V1 |
| 可选 HCM 与 CJJ 双口径 | HCS + 国标 | V1.1 |
| 环岛模板 | SIDRA | V2+ |
| 路网/绿波 | TRANSYT/RoadGee band | V2 |
| 开源仿真导出 | SUMO/Vissim CSV | V1.1–V2 |
| 断面组件市场（开源贡献） | StreetMix 心智 | V2 |

## 4. 竞争定位重述

```
轻量示意工具 -------- Crossdraw -------- 专业仿真/CAD
(draw.io)            参数化出图+规范      (Vissim/Civil)
```

Crossdraw 站在中间：**比示意工具懂交通，比仿真/CAD 更轻、更快出方案图。**

## 5. 风险

- 功能面铺太大会变成「弱版 Synchro+弱版 RoadGee」  
- 对策：路线图锁版本；分析与绿波模块化
