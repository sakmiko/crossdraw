# 交叉口出图与配时优化 · 竞品/论文调研（2026-07-16）

调研日期：2026-07-16  
调研目标：为 Crossdraw 明确「有理有据的自动优化 + 干净出图 + 工程痛点工具」优先级；对标竞品与经典方法，形成可实施清单。  
**用户优先级：** 调研与功能规划优先；一键优化/净图出图 **中优先级、稍后落地**；**暂停布局精调**。

## 关键问题
1. 竞品在交叉口设计、配时、绿波、评价、导出上的刚需能力？
2. 工程实践中优化交叉口时的典型痛点？
3. 教材/国标/论文中哪些方法可在本地工程工具落地（可审计、非黑箱）？
4. 出图哪些文字必须去掉（用户：图上不要多余文字）？

## 发现

### 1. 竞品能力画像（公开信息 + 既有审计）

| 产品 | 形态 | 强项 | 交叉口相关能力 | 与 Crossdraw 差异 |
|------|------|------|----------------|-------------------|
| **RoadGee** | Web SaaS 扣次 | 渠化+流量+信号+评价一站式；Webster；绿波多算法；Vissim 一键；出图下载 | 见 `01-roadgee-feature-audit.md` | 收费/水印/闭源；我们本地免费开源、无水印 |
| **Synchro / Tru-Traffic** | 桌面商业 | 配时优化、协调、HCM 报告、仿真联动 | 周期/绿信比/相位差优化成熟 | 昂贵；非开源；我们只做工程示意级 |
| **PTV Vissim / Visum** | 微观/宏观仿真 | 轨迹仿真、评价、信号控制接口 | 仿真验证而非出图设计 | 我们只做开放 XML/CSV 交换，非专有 inpx |
| **SIDRA / HCS** | 分析软件 | 通行能力与延误模型深 | 强分析弱 CAD 出图 | 我们偏「设计出图 + 示意评价」 |
| **TransModeler 等** | 仿真 | 网络仿真 | 与单点出图弱相关 | 非目标 |
| **开源 SUMO** | 仿真 | 可脚本化信号 | 无专业渠化 CAD | 可作后续联动，非当前 |

**RoadGee 已公开且 Crossdraw 应对齐的刚需（摘录）：**
- 渠化全参数 + 多方案；流量矩阵/大车/PHF/饱和流
- 信号相位/搭接/Webster 自动配时/生成清空
- 评价：饱和度·延误·排队·LOS；多方案对比；表导出
- 绿波：经典/优化扫描/图解/MaxBand 类；时距图
- 导出：图片、Vissim、报告

### 2. 工程现场真实痛点（设计/交管课程与方案实践）

| 痛点 | 表现 | 工具应提供 |
|------|------|------------|
| 改一处不同步 | 改流量图不变、改相位轴不对 | **同源**（表↔图↔评价） |
| 周期与相位不闭合 | Σ(G+Y+AR)≠C；双环屏障不平 | 闭合校验 + 一键闭合/平衡 |
| 关键流与 Y 不清 | 不知哪股流定周期 | Y 分解表 + 关键进口标注 |
| 绿波靠手调 | 相位差试错、多走廊打架 | 连续相位差 + 多走廊批量优化 + 带宽报告 |
| 冲突漏检 | 放行与行人/对向冲突 | 冲突矩阵/看板（规则检查） |
| 出图不专业或太吵 | 水印、脚注、说明字糊在图上 | **净图导出**（仅几何/数值，无脚注） |
| 方案比选说不清 | 多个配时说「好像更好」 | 记分卡 + 基准 Δ（延误/v/c/LOS） |
| 仿真交接难 | 只有图没法进 VISSIM | 开放交换包（诚实：非专有二进制） |
| 斜交/Y/五路难画 | 模板少、角点乱 | 多类型模板 + 角点审查 |
| 优化黑箱 | 一键后不知依据 | 方法写清 Webster/HCM/弧相交，可导出 MD 说明 |

### 3. 可落地的优化方法（论文/教材/规范 · 有依据）

| 方法 | 依据 | Crossdraw 状态 | 建议 |
|------|------|----------------|------|
| **Webster 最优周期** | Webster 1958；教材《交通管理与控制》 | 已有 `optimizeSignalTiming` / autoTimingPack | 保持；一键全方案时调用 |
| **按 y 分绿** | 同上；临界流比 | 已有 | 展示 Y 分解 |
| **HCM 延误/v/c/LOS** | HCM 信号交叉口；CJJ/T 141 服务水平思路 | 工程近似已有 | 标明近似，非完整 HCM 软件 |
| **连续相位差** | 行程时间模 C 递推 | `progressiveOffset` 已有 | 一键全方案默认先写相位差 |
| **绿波带宽 / 时距图** | 干道协调教材；Little MAXBAND 思想 | 弧相交 + 离散 MAXBAND 搜索 | 不宣称 MIP |
| **多走廊批量优化** | 工程多干线协调实践 | `optimizeAllCorridors` | 报告包已有 |
| **协调指数** | 带宽比+上下行均衡合成 | 工程合成分 A–F | 标明非国标 |
| **双环平衡** | NEMA 屏障思路示意 | dualRing + balance | 非完整控制器 |
| **冲突规则检查** | 相位放行冲突表 | conflict board | 非轨迹仿真 |
| **TWSC/环岛示意** | HCM 无信号章节思路 | 部分 | 保持示意边界 |

**不宜过度宣称：** 商业 MAXBAND-MIP、完整 NEMA dual-entry、PTV 专有 .inpx 位兼容、完整 SIDRA 容量库。

### 4. 出图规则（用户硬要求）

| 允许出现在图上 | 不允许（多余文字） |
|----------------|--------------------|
| 几何线划、车道、导向箭头 | 「同源」「示意」「非国标」脚注 |
| 短数字：流量、G/Y、λ、C | 长说明句、水印、试用 |
| 必要方向名/相位短名 | 大段标题栏说明（可用单独 MD） |
| 可选：净图模式下无图框字 | 比例尺旁长文「比例尺（示意）」 |

**策略：**  
- 交互画布：继续净几何（v0.5.74+）  
- 导出分两档：`clean-*` 净图 vs 带图框工程稿  
- 方法说明进 **MD/报告**，不进 SVG 像素

### 5. 阶段摘要

- 竞品主轴仍是 RoadGee 工作流 + Synchro 类优化深度 + 仿真交接；Crossdraw 差异化是本地、开源、可审计、无水印。  
- 下一功能包应服务痛点：**一键全方案优化（Webster+连续相位差+多走廊）** 与 **净图出图包**，依据写进 `05-professional-basis` / 本调研。  
- 布局精调按用户指示 **降优先级**。

## 调研结论

### 关键事实
1. RoadGee 公开能力树覆盖渠化–流量–信号–评价–绿波–Vissim–出图，是最直接对标（见站内审计文档）。  
2. Webster、临界流比分绿、HCM 系延误/v/c、行程时间递推相位差、带宽弧度量是可本地实现的主流方法组合。  
3. 用户明确：图上不要多余文字；优化与出图功能要继续做但排在调研之后、中优先级。
